3 · Stage 2 — Phone Verification Gate

  • Open lead in Pipeline → Lead Detail view
  • Click Verify Phone button
  • lead_extensions.phone_verified → TRUE
  • Green shield appears on pipeline card
Gate: All subsequent actions (qualify, meeting, drip, close) are blocked until phone_verified = TRUE. API returns 403.

Verify checklist

  • Red shield on pipeline card before verification
  • Green shield after verification
  • Attempting "Rapid Qualify" before verify → blocked with error toast
  • After verify → qualify button becomes active

7 · Stage 5B — Physical Meeting & Sales Handoff Stage: physical_meeting_scheduled

  • Schedule physical meeting: date, location
  • System auto-assigns salesperson (round-robin by load)
  • Lead ownership transfers to salesperson
  • Stage → physical_meeting_scheduled
  • Salesperson must upload meeting summary within 6h (SLA)
  • Quotation must be created within 12h (SLA)
SLA Breach: Missing summary or quote after deadline sets CRM lock, blocking all further actions until resolved.

Verify checklist

  • Lead disappears from telecaller's pipeline
  • Lead appears under salesperson's assigned leads
  • SAP probability → 65%
  • SLA timer visible on meeting detail
  • Upload summary → SLA timer stops
  • Confirm decision timeline checkbox works

8 · Stage 6 — Quotation Stage: quotation_sent

Create quotation

  • Quotation Builder → Add line items from SAP item list
  • Optionally apply discount (manager approval if >15%)
  • Preview PDF (opens in new tab)
  • Send via WhatsApp → stage moves to quotation_sent

Discount approval flow

DiscountAction
≤ 15%Auto-approved, can send immediately
> 15%Sent for manager approval → managers notified
ApprovedRequester notified, can send
RejectedRequester notified, must revise

WhatsApp delivery flow

  • PDF generated and sent to lead's WhatsApp
  • Status: sent → delivered (on webhook)
  • Follow-up tasks created only after delivered
  • If failed → "Retry Send" button appears on quotation card

Verify checklist

  • Items load from SAP (live prices)
  • PDF preview opens correctly
  • WhatsApp send triggers status = sent
  • Discount >15% shows pending approval badge
  • Retry button appears on failed send
  • Stage → quotation_sent after successful send

10 · Special Flows

No-Response Flow (4+ failed calls)

  • Log 4+ call attempts with outcome = no_response
  • Lead appears in No-Response queue tab
  • Recovery WhatsApp option becomes available
  • Send recovery WhatsApp message to re-engage
Check: No-Response queue badge count updates after 4th failed attempt.

Drip Campaign

  • From full_qualified lead, click "Enter Drip"
  • Select track: 1_month / 3_month / 6_plus_month
  • Drip engine sends WhatsApp messages on schedule
  • Lead appears in Drip Queue tab with next message time
  • Manual exit: "Exit Drip" with reason
Cadence: 1-month = 9 messages over 17 days. 6-month = 13 messages over 168 days.

6-Month Funnel / Reactivation

  • Set timeline = 6_plus_months OR reactivation_flag = true on LOST
  • Lead appears in "6+ Month Funnel" queue
  • Set reactivate_by date on lead
  • Daily scanner sends reminder 7 days before reactivate_by
Notification badge shows when a reactivation reminder fires.

Idle Lead Scanner

  • Leads with no activity for 14+ days → Idle Queue
  • Yellow warning banner on lead detail after 14 days idle
  • Idle queue badge shows count

Sales Hand-Back

  • Sales can return a lead to telecaller with a reason
  • Lead stage → handed_back, appears in Reactivation queue
  • SAP probability reset to 10%

11 · Manager Features

Analytics Dashboard (Manager view)

  • Login as manager → Dashboard shows team-wide data
  • Team leaderboard with per-telecaller breakdown
  • Total leads, calls, meetings, WON/LOST across all users
  • Recent activity includes "actor" name
Telecaller login → sees only own data. Manager login → sees entire team.

Discount Approvals

  • Telecaller requests discount >15% → all managers notified
  • Manager sees pending approval in notification bell
  • Approve → requester notified, quote can be sent
  • Reject → requester notified with reason

Notification Bell

  • Unread count badge on bell icon
  • Mark individual/all as read
  • Types: discount_approval_request, reactivation_reminder, sla_breach
